# 05 技术方案

> 轻量路径把命令参数和数据结构写在本文，不单独保留 `docs/07-api-spec.md`。

## 1. 技术栈与版本

- 语言：Python 3.11
- CLI：标准库 `argparse`
- 路径处理：标准库 `pathlib`
- 测试：标准库 `unittest` 或 pytest（待确认；Phase1 可先用标准库）

## 2. 关键技术决策

- 清理规则实现为纯函数：`clean_text(text: str) -> str`。
- CLI 层负责读取输入文件、调用清理函数、写入输出文件。
- 输出文件默认覆盖；如需防覆盖保护，进入 Phase2 再评估。
- 不保留 `docs/07-api-spec.md`：本项目不是稳定对外 API，命令契约在本文和 08/09 中说明即可。

## 3. Phase 技术约束

- 允许：标准库、同步处理、单文件输入输出。
- 禁止：第三方 CLI 框架、数据库、Web 服务、AI 模型调用。

## 4. 命令参数约定

```bash
python -m backend.cli --input raw.txt --output clean.txt
```

- `--input`：必填，输入文本文件路径。
- `--output`：必填，输出文本文件路径。
- 成功退出码：`0`
- 参数错误或输入文件不存在：非 `0`

## 5. 清理规则

- 去掉每行首尾空白。
- 连续两个及以上空行合并为一个空行。
- 行首项目符号 `*` 或 `+` 统一为 `-`。
