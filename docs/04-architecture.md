## 最小示例（填写粒度参考）

```markdown
## 1. 整体架构图
用户 → CLI / Web UI → 应用服务 → 领域处理模块 → 输出结果

## 2. 模块划分
| 模块 | 阶段 | 状态 | 职责 | 对应功能 |
|---|---|---|---|---|
| 输入适配层 | [P1] | P1-已设计 | 接收用户输入并做基础校验 | F-001 |
| 领域处理模块 | [P1] | P1-已设计 | 执行核心业务规则 | F-001 |
| 历史记录模块 | [P2] | 骨架 | 保存与查询处理历史 | F-002 |

## 3. 技术选型理由
- 采用单体结构：Phase1 功能边界小，避免过早拆分服务。
- 暂不引入队列：Phase1 无异步批处理需求。

## 4. 部署 / 运行拓扑约束
- Phase1 本机单机运行：CLI / Web UI 与应用服务运行在同一台开发机。
- 远程依赖：若本机无法承载大模型或重型数据库，Phase1 使用 Mock / 远程 API，完整能力列入服务器资源预案。
- 端口与网络：本机端口、联网范围和公司服务器使用权限以 `docs/env/local-env.md` 的人工确认项为准。
```
