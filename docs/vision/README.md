# docs/vision/ 产品愿景与叙事

> 本目录是**整理后愿景区**：放经输入材料评审、补齐和确认后的产品愿景叙事。普通用户的原始材料默认先放 `docs/inputs/`；AI 评审其 Product Vision 就绪度后，再生成或更新本目录中的愿景文档。

## 放什么

- 产品愿景、完整产品图景、业务叙事（如 `product-vision.md`）。
- 从 `docs/inputs/` 提炼并保留来源锚点的市场背景、用户故事、竞品叙事摘要。
- 旧项目已有且仍作为上游锚点的成熟愿景文档。

## 不放什么

- 已确认的需求 / 设计 / 计划——那些是 AI 输出，放 `docs/00-09` 或 `docs/design/`。
- 决策记录 → `docs/decisions/`；调研 → `docs/research/`；会议 / 访谈 → `docs/meetings/`。
- 尚未归类、未经评审的原始输入 → `docs/inputs/`。

## 与文档体系的关系

- `docs/` 的输入 / 输出区分、分区规则、裁剪：见 `docs/README.md`。
- `docs/inputs/` 如何经愿景就绪评估进入 product-vision，再驱动需求链：见 `ai/document-lifecycle-rules.md` §3。
