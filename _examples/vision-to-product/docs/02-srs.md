# 02 需求规格说明（SRS）

> Phase1 需求必须可验证；后续阶段只保留粗粒度，待升阶段时细化。

## Phase1 需求

| ID | 阶段 | 状态 | 需求 | 验证方式 |
|---|---|---|---|---|
| REQ-001 | [P1] | P1-已设计 | 系统必须提供新增知识条目的能力，输入标题与正文后保存为本地条目。 | 新增后可在列表中看到该条目 |
| REQ-002 | [P1] | P1-已设计 | 系统必须为每条知识生成不超过 120 字的摘要；Phase1 可使用规则摘要或 mock 摘要。 | 新增条目后摘要字段非空且长度受控 |
| REQ-003 | [P1] | P1-已设计 | 系统必须支持按关键词搜索标题、摘要和正文。 | 输入命中关键词后返回对应条目 |
| REQ-004 | [P1] | P1-已设计 | 系统必须支持查看单条知识详情，包含标题、摘要、正文、创建时间。 | 根据条目 ID 可查看完整详情 |
| REQ-005 | [P1] | P1-已设计 | 系统必须把知识条目持久化到本地 SQLite，工具重启后数据不丢失。 | 新增条目后重启工具仍可查询 |
| REQ-006 | [P1] | P1-已设计 | 系统必须提供 CLI 命令入口，支持 add、search、show 三类操作。 | CLI 命令参数与输出符合 `docs/07-api-spec.md` |
