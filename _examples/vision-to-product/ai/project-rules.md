## 1. Phase边界

当前阶段：Phase1

允许：
- 本地单人使用
- CLI 命令入口
- SQLite 本地持久化
- 使用可替换的摘要服务接口；Phase1 可先用规则摘要或 mock 摘要实现
- 关键词搜索、列表查看、详情查看

禁止：
- 登录、多用户、云同步
- 向量数据库、语义检索、复杂 Agent 编排
- 标签体系、Markdown 导出、多项目知识库
- 后端 Web 服务、浏览器前端
- 未经确认引入第三方 AI SDK 或在线模型依赖

下一阶段预告：
- Phase2：标签、多项目知识库、Markdown 导出、可选语义检索

## 2. 技术栈约束

- 确定：Python 3.11 / SQLite（标准库 `sqlite3`）/ 标准库 `argparse` / 标准库 `pathlib`
- 摘要能力：Phase1 默认规则摘要或 mock 摘要；接入真实模型前必须人工确认供应商、密钥管理与费用风险
- 禁止引入：Web 框架、ORM、向量数据库、后台任务队列、未经确认的 AI SDK

## 5. 编码约定与禁区

> Phase 级功能禁止见 §1，技术栈替代品禁止见 §2，本节只管代码层。
> 每条尽量具体可执行；没有则写“无”，不要留空占位。

### 5.1 既有约定（新代码必须向其看齐）
- 命名：Python 文件、函数、变量使用 snake_case；类名使用 PascalCase
- 分层与目录：CLI 参数解析放 `backend/cli.py`；业务逻辑放 `backend/service.py`；数据访问放 `backend/repository.py`；数据库初始化放 `backend/db.py`
- 既有模式：CLI 层不直接写 SQL；所有数据库读写统一经 repository 层
- 错误处理 / 日志：CLI 参数错误返回非 0 退出码；用户可理解错误写 stderr；Phase1 不引入日志框架

### 5.2 禁区（未经人工确认不得触碰）
- 不得擅改的文件 / 模块：`ai/` 通用规则文件、数据库迁移 / 初始化逻辑
- 不得擅自引入的依赖：任何新依赖须先确认，尤其 AI SDK、ORM、Web 框架
- 不得自行实现的功能：登录、多用户、云同步、向量检索、标签、导出、Web UI
- 愿景 / 01 中的功能点不等于已批准实现；阶段归属以 `docs/03-prd.md` §3 路线图为准，编码以 §1 当前阶段为准

## 6. AI修改确认规则

- AI在进行任何文件新增、修改、删除、重命名、格式化、批量替换前，必须先说明计划、影响范围与预期变更文件，并等待用户明确确认后再执行。
- AI在运行任何可能写入文件、安装依赖、生成构建产物、修改配置或改变项目状态的命令前，必须先询问用户确认。
- 只读分析操作（如读取文件、搜索代码、查看 Git 状态）无需逐次确认，但不得借只读分析之名修改项目内容。
- 用户在单次消息中明确要求“直接修改”“执行修复”“不必确认”等同类授权时，仅对该次明确任务生效；后续新任务仍默认先确认再修改。
