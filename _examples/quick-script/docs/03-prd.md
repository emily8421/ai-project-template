# 03 产品需求文档（PRD）

> 轻量项目也需要明确范围；本文只写当前小闭环，不扩展愿景功能。

## 1. 功能范围

- [P1][P1-已设计] F-001：读取输入文本文件；对应 REQ-001。
- [P1][P1-已设计] F-002：执行会议纪要格式清理；对应 REQ-003~REQ-005。
- [P1][P1-已设计] F-003：写入输出文本文件；对应 REQ-002。
- [P1][P1-已设计] F-004：处理参数缺失和输入文件不存在错误；对应 REQ-006。
- [P2][骨架] F-101：更多规则、行动项提取；对应 REQ-101~REQ-102。
- [愿景][骨架] F-201：结构化 Markdown 导出；对应 REQ-201。

## 2. 优先级

- P0：F-001、F-002、F-003，形成脚本可用闭环。
- P1：F-004，保证错误时可理解。
- P2：F-101、F-201，后续阶段再细化。

## 3. Phase 划分

- Phase1：单输入文件、单输出文件、三条固定清理规则、基础错误处理。
- Phase2：可配置规则、行动项提取。
- 愿景阶段：结构化 Markdown 导出。

## 4. 非目标（明确不做）

- Phase1 不做数据库、Web UI、HTTP API。
- Phase1 不做 AI 摘要或行动项提取。
- Phase1 不做批量目录处理或配置文件系统。
