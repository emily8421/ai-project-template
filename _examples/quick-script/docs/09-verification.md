# 09 验证计划（Verification）

> 小脚本也要定义“怎么算对”。本文覆盖 Phase1 的全部 REQ。

## 1. 测试策略

- 单元测试：直接测试 `clean_text(text: str) -> str` 的清理规则。
- CLI 测试：使用临时输入 / 输出文件验证命令行为。
- 错误测试：验证缺参数、输入文件不存在时的退出码和错误提示。
- 人工验收：用一份真实会议纪要样例运行脚本，确认输出适合分享。
