# 09 验证计划（Verification）

> 本文回答“怎么算对”。与 `docs/08-dev-plan.md` 正交：08 说明何时做，09 说明如何验收。

## 1. 测试策略

- 单元测试：覆盖摘要模块、repository 查询、新增和详情读取。
- 集成测试：通过 CLI 命令验证 add/search/show 的端到端行为。
- 持久化测试：使用临时 SQLite 文件，新增后重新初始化连接再读取。
- 边界测试：空标题、空正文、无匹配关键词、不存在 ID。
- 人工验收：按 CLI 示例执行命令，确认输出可读且符合契约。
