# 01 用户需求说明

> 从愿景全量抽取。需求带阶段标签，但阶段归属最终以 `docs/03-prd.md` §3 为准。

- [P1][P1-已设计] 用户希望保存一段知识文本，并给它一个标题。
- [P1][P1-已设计] 用户希望系统为知识文本生成简短摘要，便于快速回顾。
- [P1][P1-已设计] 用户希望通过关键词搜索已保存的知识条目。
- [P1][P1-已设计] 用户希望查看某条知识的标题、摘要、原文和创建时间。
- [P1][P1-已设计] 用户希望数据保存在本地，关闭工具或重启后不丢失。
- [P2][骨架] 用户希望给知识条目添加标签。
- [P2][骨架] 用户希望按项目或主题组织多个知识库。
- [P2][骨架] 用户希望把知识条目导出为 Markdown。
- [愿景][骨架] 用户希望通过语义检索找回概念相关但关键词不同的内容。
- [愿景][骨架] 用户希望未来可接入更智能的 AI 助手，但不影响 Phase1 的简单闭环。
