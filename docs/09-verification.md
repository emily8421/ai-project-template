# 09 验证计划（Verification Plan）

> **文档定位**：定义需求如何验证、用例如何覆盖、资源如何确认、验收结果如何记录。本文回答“怎么算对”。
>
> **适用条件**：所有项目必备。即使是 Lean / 小工具项目，也必须保留最小验证计划。
>
> **上游输入**：`docs/02-srs.md`、`docs/03-prd.md`、`docs/04-architecture.md`、`docs/05-tech-spec.md`、`docs/08-dev-plan.md`、`docs/env/local-env.md`。

> **下游输出**：约束测试实现、人工验收、CI、发布检查和 Sprint 总结。
