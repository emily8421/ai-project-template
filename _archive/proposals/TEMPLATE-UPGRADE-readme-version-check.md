# TEMPLATE-UPGRADE — sync 后校验根 README 模板版本号一致性

> 去项目化提案，目标回流到 `ai-project-template` 模板仓库。本文件不写入任何具体项目业务、账号或路径。

## 动机

`sync-template.*` 只同步 `template-sync.json` 定义的方法论文件清单；根 `README.md` 属于受保护的项目专属文件，同步流程不会触碰它。`check-derived-sync.sh` 也仅校验「同步提交未越界改项目专属文件」，**不校验 README 中模板版本号声明是否与 `VERSION` 一致**。

后果：派生项目根 README 中常见的「派生自 ai-project-template vX.Y.Z」「当前已同步至模板 vX.Y.Z」等声明完全依赖人工维护，sync 后极易滞后且无任何提示。实测：某派生项目从 v1.7.0 连续同步到 v1.21.0（跨多个版本），README 仍标 v1.7.0，直至同步后整理审计才发现。

## 拟改

在派生项目同步边界检查脚本 `scripts/check-derived-sync.sh`（与 `.ps1` 入口行为一致）中，新增一项**非阻断**的模板版本号一致性告警：

1. 读取根 `VERSION`，得到当前模板版本（如 `v1.21.0`）。
2. 扫描根 `README.md` 中显式的模板版本声明，就近取最「当前」语义的声明，候选匹配模式：
   - `当前已同步至模板 vX.Y.Z` / `当前.*同步.*vX.Y.Z`
   - `派生自 ai-project-template（vX.Y.Z）` / `派生自 … vX.Y.Z`
3. 若 README 存在此类「当前 / 已同步」语义的版本声明且与 `VERSION` 不一致，输出告警行（如 `⚠ README 模板版本声明 vX.Y.Z 与 VERSION vX.Y.Z 不一致，请人工核对`），**不计入 `FAILURES`、不阻断**检查通过。
4. 若 README 未含任何模板版本声明，不告警（README 不强制写版本号）。

设计为非阻断告警而非硬失败的理由：README 可能合法地包含历史性 / 叙事性版本引用（如变更记录「2026-06 曾同步至 v1.7.0」），硬阻断会误伤。告警 + 提示人工核对是正确粒度。

可选增强：`sync-template.*` 在 commit 模式完成后，尾部输出一行「请核对根 README 的模板版本号声明」。

## 版本

建议纳入下一个模板小版本（具体版本号由模板维护者定）。仅新增一项告警检查，不改变既有检查语义与退出码。

## 影响

- 受影响：所有派生项目的 `scripts/check-derived-sync.*`（随模板下行同步刷新）。
- 兼容性：非破坏性。既有「边界检查通过 / 失败」语义与退出码不变，仅新增告警行。
- 实现成本：低（README 正则扫描 + 与 `VERSION` 字符串比对）。
- 不影响：`sync-template` 同步清单、受保护项目文件判定、提交信息格式校验等既有逻辑。

## 待模板维护者确认

- 告警 vs 阻断：本提案建议告警（非阻断）；若倾向阻断，需先约定 README 版本声明的规范写法，避免误伤叙事性引用。
- 匹配模式：本提案给出候选正则，最终模式由模板维护者按 README 模板约定收敛。
