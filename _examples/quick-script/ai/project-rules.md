## 0. 项目标识

项目名称：Meeting Notes Cleaner
代号/缩写：mnc

## 1. Phase边界

当前阶段：Phase1

允许：
- Python 3.11 本地 CLI 脚本
- 单个输入文件 → 单个输出文件
- 同步处理
- 标准库实现

禁止：
- 数据库存储
- Web UI / HTTP API
- AI 模型调用
- 批量目录处理
- 自动提取行动项、标题识别、复杂 Markdown 结构化

下一阶段预告：
- Phase2：更多清理规则、行动项提取、Markdown 导出增强

## 2. 技术栈约束

- 确定：Python 3.11 / 标准库 `argparse` / 标准库 `pathlib`
- 禁止引入：Click、Typer、FastAPI、Flask、任何数据库驱动、任何 AI SDK

## 2.5 运行环境与资源约束

- 本机环境文档：示例项目不采集真实机器信息；派生项目需运行 `scripts/collect-env.ps1` 生成 `docs/env/local-env.md`。
- Demo 阶段必须能在本机运行的部分：CLI 文本清理脚本、单文件输入输出、单元测试。
- 允许降级 / Mock / 远程运行的部分：无；Phase1 不需要远程资源。
- 禁止在本机运行的重资源部分：数据库、Web 服务、AI 模型调用。
- 是否允许使用公司服务器：不需要。
- 若需服务器，资源申请口径：暂不需要；若 Phase2 扩展批处理大文件，再评估 CPU、内存与磁盘 I/O。
## 3. 项目形态与文档裁剪

- 是否有持久化存储：无
- 是否有对外接口：无（仅本地脚本命令；命令参数在 `docs/05-tech-spec.md` 与 `docs/08-dev-plan.md` 描述，不单独保留 07）
- docs/06-db-design.md：省略
- docs/07-api-spec.md：省略
- 需要保留的代码目录：backend/ tests/

按项目形态裁剪说明：

- 轻量本地脚本不提供稳定外部接口契约 → `docs/07-api-spec.md` 省略
- 仅保留 `docs/00-05`、`docs/08-dev-plan.md`、`docs/09-verification.md`

## 4. 目录规范的项目特例

轻量脚本项目不创建 frontend/、docker/；仅保留 backend/ 与 tests/。

## 5. 编码约定与禁区

> Phase 级功能禁止见 §1，技术栈替代品禁止见 §2，本节只管代码层。

### 5.1 既有约定（新代码必须向其看齐）
- 命名：Python 文件、函数、变量使用 snake_case
- 分层与目录：`backend/cleaner.py` 放清理规则；`backend/cli.py` 放参数解析；测试放 `tests/test_cleaner.py`
- 既有模式：清理函数接收字符串并返回字符串，文件读写只在 CLI 层处理
- 错误处理 / 日志：文件不存在或参数错误时输出用户可理解错误并返回非 0 退出码；Phase1 不引入日志框架

### 5.2 禁区（未经人工确认不得触碰）
- 不得擅改的文件 / 模块：`ai/` 通用规则文件
- 不得擅自引入的依赖：任何新依赖须先确认
- 不得自行实现的功能：AI 摘要、行动项提取、Web 服务、数据库缓存、批量目录处理
- 愿景 / 01 中的功能点不等于已批准实现；阶段归属以 `docs/03-prd.md` §3 路线图为准，编码以 §1 当前阶段为准

## 6. AI修改确认规则

- AI在进行任何文件新增、修改、删除、重命名、格式化、批量替换前，必须先说明计划、影响范围与预期变更文件，并等待用户明确确认后再执行。
- AI在运行任何可能写入文件、安装依赖、生成构建产物、修改配置或改变项目状态的命令前，必须先询问用户确认。
- 只读分析操作（如读取文件、搜索代码、查看 Git 状态）无需逐次确认，但不得借只读分析之名修改项目内容。
- 用户在单次消息中明确要求“直接修改”“执行修复”“不必确认”等同类授权时，仅对该次明确任务生效；后续新任务仍默认先确认再修改。
