# docs/vision/ 产品愿景与叙事

> 本目录是**人工输入区**：放你提供的产品愿景、叙事、业务背景等输入材料。它们是工程文档的**原料**，不直接驱动开发——必须先经评审（`ai/prompts/docs/01-review-inputs.md`）再进入 `docs/00-09`。

## 放什么

- 产品愿景、完整产品图景、业务叙事（如 `product-vision.md`）。
- 市场背景、用户故事、竞品叙事等输入材料。

## 不放什么

- 已确认的需求 / 设计 / 计划——那些是 AI 输出，放 `docs/00-09` 或 `docs/design/`。
- 决策记录 → `docs/decisions/`；调研 → `docs/research/`；会议 / 访谈 → `docs/meetings/`。
- 尚未归类的原始输入 → `docs/inputs/`。

## 与文档体系的关系

- `docs/` 的输入 / 输出区分、分区规则、裁剪：见 `docs/README.md`。
- 愿景如何驱动需求链（Vision-first 入口）：见 `ai/document-lifecycle-rules.md` §3。
