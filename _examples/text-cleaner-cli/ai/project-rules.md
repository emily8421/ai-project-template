## 0. 项目标识
项目名称：TextCleaner
代号/缩写：tc

## 1. Phase边界
当前阶段：Phase1

允许：
- Python 3.11 本地 CLI
- 单文件输入、单文件输出
- 同步处理

禁止：
- 数据库存储
- HTTP API / Web UI
- 批处理队列、目录级批量任务

下一阶段预告：
- Phase2：支持多种清洗规则组合与批量目录处理

## 2. 技术栈约束
- 确定：Python 3.11 / 标准库 `argparse` / 标准库 `pathlib`
- 禁止引入：Click、Typer、Flask、FastAPI、任何数据库驱动

## 3. 项目形态与文档裁剪
- 是否有持久化存储：无
- 是否有对外接口：有（CLI）
- docs/06-db-design.md：省略
- docs/07-api-spec.md：保留
- 需要保留的代码目录：backend/ tests/ scripts/

## 4. 目录规范的项目特例
无，遵循 global-rules 通用目录标准。

## 5. 编码约定与禁区

> Phase 级功能禁止见 §1，技术栈替代品禁止见 §2，本节只管代码层。

### 5.1 既有约定（新代码必须向其看齐）
- 命名：Python snake_case（函数/变量/文件）；类 PascalCase
- 分层与目录：`backend/` 下分 `cli/`（参数解析与命令入口）和 `service/`（文本清洗逻辑）
- 既有模式：文本处理逻辑放 `service` 层，CLI 层只负责参数和文件读写
- 错误处理 / 日志：用户输入错误返回非零退出码并输出简短报错；Phase1 不引入日志框架

### 5.2 禁区（未经人工确认不得触碰）
- 不得擅改的文件 / 模块：`scripts/` 中的发布脚本、`ai/` 通用规则文件
- 不得擅自引入的依赖：任何新依赖须先确认
- 不得自行实现的功能：Web 服务、数据库缓存、后台任务——均为 §1 禁止项
