# 03 产品需求文档（PRD）

## 1. 功能范围
- 待办 CRUD（创建/查询/更新完成状态/删除）—— 对应 REQ-1~4，均在当前 Phase1 内
- 本地持久化 —— 对应 REQ-5

## 2. 优先级
- P0：创建、查询、标记完成、删除、持久化（全部进 Phase1）

## 3. Phase 划分
- Phase1（当前）：[交付物形态：Demo] 单人本地 CRUD + SQLite 持久化；退出标准：本机 API 可演示待办增删改查闭环
- Phase2（预告）：[交付物形态：MVP] 分类标签、到期时间（仍单人）；退出标准：常见个人待办管理场景可真实使用

## 4. 非目标（明确不做）
- 不做用户注册/登录/鉴权（单人）
- 不做多用户、不做人共享
- 不做云端部署（Phase1 只本地）
