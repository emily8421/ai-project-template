## 0. 项目标识

代号/缩写：mt（用于表前缀；本项目仅 todos 单表，暂未使用）

## 1. Phase边界
当前阶段：Phase1

允许：
- FastAPI + SQLite
- 单人本地运行
- 同步实现

禁止：
- 多用户 / 用户系统 / 鉴权
- PostgreSQL / 其它独立数据库服务
- 云端部署

下一阶段预告：
- Phase2：分类标签、到期时间（仍单人）

## 2. 技术栈约束
- 确定：Python 3.11 / FastAPI 0.110+ / SQLite（标准库 sqlite3）
- 禁止引入：Flask、Django、SQLAlchemy 等 ORM、PostgreSQL 驱动

## 3. 项目形态与文档裁剪

## 4. 目录规范的项目特例
无，遵循 global-rules 通用目录标准。

## 5. 编码约定与禁区

> Phase 级功能禁止见 §1，技术栈替代品禁止见 §2，本节只管代码层。

### 5.1 既有约定（新代码必须向其看齐）
- 命名：Python snake_case（函数/变量/文件）；类 PascalCase
- 分层与目录：backend/ 下分 api/（路由）、model/（数据访问）、db.py（连接与建表）；HTTP 处理只在 api 层
- 既有模式：数据访问统一经 model 层的 repo 函数，api 层不直接写 SQL
- 错误处理 / 日志：资源不存在返回 404；入参不符返回 422；Phase1 不引入全局异常中间件

### 5.2 禁区（未经人工确认不得触碰）
- 不得擅改的文件 / 模块：backend/db.py（建表逻辑）；global-rules.md / index.md（通用层，永不动）
- 不得擅自引入的依赖：任何新依赖须先确认（Phase1 目标是零额外依赖）
- 不得自行实现的功能：鉴权、多用户、定时任务——均为 §1 禁止项，点名为禁区
