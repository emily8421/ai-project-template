# 产品愿景叙事：MiniTodo

> 产品愿景叙事·不直接驱动开发。本文是工程文档的输入，AI 需要先从中抽取 `docs/00-09`，再进入开发。

我希望做一个极简的本地待办清单 API，用来练习和验证一个有数据库、有对外接口的小项目如何从需求走到设计、开发计划和验收闭环。

第一阶段只需要支持单人本地使用。用户可以创建待办项、查看待办列表、修改完成状态、删除待办项，并且这些数据需要保存到本地 SQLite，服务重启后不丢失。

这个项目不需要前端页面，不需要登录、多用户、权限管理、云部署或复杂 ORM 抽象。重点是把 REST API、数据库表设计和验证用例之间的关系写清楚，让后续实现可以小步完成、容易检查。

未来也许可以支持截止日期、优先级、标签、多用户和前端界面。但这些都不是第一阶段目标，Phase1 只做最小可验证的 CRUD API。
