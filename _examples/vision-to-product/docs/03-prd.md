# 03 产品需求文档（PRD）

> 依据 `docs/00-02` 生成，人工确认。§3 是阶段标签的唯一来源。

## 1. 功能范围

- [P1][P1-已设计] F-001：新增知识条目，输入标题与正文并保存；对应 REQ-001。
- [P1][P1-已设计] F-002：生成知识摘要；对应 REQ-002。
- [P1][P1-已设计] F-003：按关键词搜索标题、摘要和正文；对应 REQ-003。
- [P1][P1-已设计] F-004：查看知识条目详情；对应 REQ-004。
- [P1][P1-已设计] F-005：本地 SQLite 持久化；对应 REQ-005。
- [P1][P1-已设计] F-006：CLI 命令入口；对应 REQ-006。
- [P2][骨架] F-101：标签管理、项目知识库、Markdown 导出；对应 REQ-101~103。
- [愿景][骨架] F-201：语义检索和智能助手；对应 REQ-201。

## 2. 优先级

- P0：F-001、F-003、F-004、F-005、F-006，构成可运行知识管理闭环。
- P1：F-002，摘要能力必须存在，但 Phase1 可用规则摘要或 mock 摘要，不强依赖在线模型。
- P2：F-101、F-201，进入后续阶段再细化。

## 3. Phase 划分

- Phase1：本地单人 CLI + SQLite；支持新增、摘要、搜索、详情查看和持久化。
- Phase2：标签、多项目知识库、Markdown 导出。
- 愿景阶段：语义检索、智能助手、更多 AI 能力。

## 4. 非目标（明确不做）

- Phase1 不做登录、多用户、云同步。
- Phase1 不做 Web UI、HTTP API 或后台服务。
- Phase1 不接入向量数据库或语义检索。
- Phase1 不默认接入真实在线模型；如需接入，必须先确认供应商、密钥管理和费用风险。
