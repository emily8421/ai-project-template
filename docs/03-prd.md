# 03 产品需求文档（PRD）

> AI生成初稿，人工确认。依据 00-02 生成，明确功能范围、优先级、Phase划分与交付物形态。
> 写法：§3 阶段路线图 = 阶段标签唯一来源；AI 在此提议路线图，人工确认。每个 Phase 必须同时声明功能范围、交付物形态（Demo/MVP/产品）和进入/退出标准。按 global-rules.md §8 积累式演进。

## 1. 功能范围
（逐条列出本项目要做的功能，标注对应的 02-srs REQ 编号；区分“当前 Phase 内”与“后续 Phase”）

## 2. 优先级
（P0 必须 / P1 重要 / P2 可选；明确哪些进当前 Phase）

## 3. Phase 划分
（每个 Phase 开放的功能 + 交付物形态 + 进入/退出标准；与 ai/project-rules.md §1 边界一致）

## 4. 非目标（明确不做）
（防止范围蔓延：本轮明确不实现的功能）

## 最小示例（填写粒度参考）

```markdown
## 1. 功能范围
- [P1][P1-已设计] F-001：用户可输入一段文本并获得规范化结果；对应 REQ-001。
- [P2][骨架] F-002：用户可保存历史处理记录；对应 REQ-005。

## 2. 优先级
- P0：F-001，当前 Phase 必须完成。
- P1：F-002，进入 Phase2 前再细化。

## 3. Phase 划分
- Phase1：[功能范围：REQ-001 / F-001][交付物形态：Demo] 输入文本 → 处理 → 返回结果；退出标准：可在本机演示核心闭环。
- Phase2：[功能范围：REQ-005 / F-002][交付物形态：MVP] 历史记录、批量处理、导出；退出标准：关键生产要素可真实使用。

## 4. 非目标（明确不做）
- Phase1 不做登录、多用户协作、云端同步。
```
