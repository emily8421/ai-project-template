# TEMPLATE-UPGRADE: AI 会话续接与断点恢复机制

> 类型：模板仓库内直接发起的模板优化提案。
> 状态：待评估 / 待落地。
> 关联：现有 `NEXT-STEPS.md` 本地续接约定、`ai/prompts/dev/02-run-task.md` 切换工具续接、`ai/global-rules.md` 工具切换说明。

## 1. 背景与问题

在模板仓库或派生项目中使用 AI CLI 处理任务时，会话经常可能被迫中断：

- token 达到上限。
- 对话长度达到上限。
- 单次会话时长达到上限。
- 误操作关闭窗口。
- 断电、重启、网络或终端异常。

当前已有 `NEXT-STEPS.md` 作为本地临时续接文件，并已通过 `.gitignore` 排除。但它依赖人工主动提醒 AI 写入：如果用户忘记要求 AI 记录，或 AI 在输出执行计划后突然中断，计划、上下文、已完成步骤与下一步动作仍会丢失。

因此需要把“会话续接 / 断点恢复”做成模板的一等机制：AI 在关键节点自动维护本地续接文件，新窗口启动时自动读取并恢复上下文，而不是依赖用户事后回忆。

## 2. 设计目标

- 自动保留任务断点：即使 CLI 突然中断，新窗口也能知道上次做了什么、做到哪儿、下一步是什么。
- 保留 AI 已输出的执行计划：计划一旦形成，就应落盘，而不是只存在聊天上下文中。
- 不污染正式项目文档：续接文件默认本地忽略，不进入模板或派生项目正式提交。
- 对模板仓库与派生项目都适用：模板维护任务、项目开发任务、文档整理任务都可复用。
- 与现有确认规则兼容：自动续接记录属于本地临时状态文件，但若规则要求写文件前确认，应在模板规则中明确该行为的授权边界。
- 简单可执行：优先用 Markdown 文件与规则约束，避免一开始引入复杂状态系统。

## 3. 建议方案

### 主推路径

短期沿用根目录：

```text
NEXT-STEPS.md
```

中期建议迁移或兼容更明确的路径：

```text
.ai/session-handoff.md
```

建议采用兼容策略：新机制优先使用 `.ai/session-handoff.md`，若项目中已存在 `NEXT-STEPS.md`，新窗口启动时也读取并提示是否迁移；模板仍继续忽略两类本地续接文件。

路径语义：`.ai/session-handoff.md` 是 AI 会话之间的本地交接记录，只用于断点恢复，不是需求、设计、任务单或验收文档。

## 4. 自动更新触发点

建议在 AI 规则中规定：只要发生以下情况，AI 应主动刷新续接文件，无需等待用户额外提醒。

- **输出执行计划后**：将计划、当前任务、待办步骤写入续接文件。
- **开始多步骤任务前**：记录任务目标、影响范围、计划修改文件与验证方式。
- **完成计划中的一个步骤后**：更新已完成 / 进行中 / 未完成状态。
- **准备修改文件前**：记录预期变更文件、修改原因与风险点。
- **完成文件修改后**：记录实际新增 / 修改 / 删除文件、验证结果与下一步。
- **遇到阻塞或待确认项时**：记录阻塞原因、待确认问题与建议选项。
- **结束回复前**：如果仍有未完成任务，刷新“下次优先做什么”。

若当前操作完全是只读问答、且没有形成可续接任务，可不更新续接文件。

## 5. 新窗口启动流程

建议把新会话启动流程固化到规则与提示词中：

1. 先读取 `ai/index.md` 及其列出的规则文件。
2. 检查本地续接文件：优先 `.ai/session-handoff.md`，其次 `NEXT-STEPS.md`。
3. 运行只读状态检查：`git status --short --branch`；必要时再看最近提交与相关 diff。
4. 向用户简述恢复到的状态：当前任务、已完成、下一步、待确认项。
5. 若续接文件与 git 状态冲突，以实际工作区状态为准，并提示人工确认。

## 6. 建议文件结构

续接文件建议使用稳定结构，便于不同 AI 工具读写：

```markdown
# AI Session Handoff

> 本文件为本地会话续接记录，不进入正式提交。

## 当前任务

一句话说明正在处理什么。

## 当前进度

- 已完成：
- 进行中：
- 未完成：

## 执行计划

1. ...
2. ...
3. ...

## 最近改动

- 新增：
- 修改：
- 删除：

## 下次优先做

1. ...
2. ...

## 阻塞 / 待确认

- ...

## 恢复命令

- `git status --short --branch`
- `git diff -- ...`
- `bash scripts/check-template.sh`
```

模板可提供样例文件，例如：

```text
template-docs/session-handoff.example.md
```

真实续接文件仍应保持本地临时，不进入正式提交。

## 7. 拟改范围

### 规则文件

- `ai/index.md`：若新增独立规则文件，需要把它列入规则入口。
- `ai/session-rules.md`（建议新增）：集中定义会话续接、自动更新触发点、新窗口恢复流程与文件格式。
- `ai/global-rules.md`：现有“切换工具续接”段落改为引用 `ai/session-rules.md`，避免规则散落。
- `ai/project-rules.md`：确认“AI 修改确认规则”与续接文件自动更新的关系；建议明确本地续接文件属于会话状态记录，但首次启用 / 写入仍需遵守项目确认规则，或由用户在任务开始时授权。

### 提示词

- `ai/prompts/dev/02-run-task.md`：执行计划生成后自动写入 / 更新续接文件；每完成一个 Sprint 步骤后刷新状态。
- `ai/prompts/dev/05-fix-bug.md`：复现、定位、修复、验证步骤应写入续接文件。
- `ai/prompts/dev/09-sprint-summary.md`：输出总结时同步更新“已完成 / 后续任务”。
- `ai/prompts/maintainers/11-template-proposal-summary.md`：模板提案汇总计划落盘，避免维护中断后丢失分析。
- `ai/prompts/maintainers/12-sync-template.md`：同步流程中记录当前同步阶段、已执行命令与下一步。
- `ai/prompts/maintainers/15-post-sync-cleanup.md`：迁移计划与待确认项落盘。
- 其他会输出多步骤计划的 prompt，可逐步补齐续接要求。

### 模板文档

- `.gitignore`：继续忽略 `NEXT-STEPS.md`，并新增忽略 `.ai/session-handoff.md`；如使用 `.ai/` 目录，还需避免误忽略未来可能正式纳入版本的 `.ai` 配置。
- `CONTRIBUTING.md`：说明本地续接文件仍不作为正式模板文档提交；长期内容应转为提案、README、SOP 或维护文档。
- `MAINTAINERS.md`：补充模板维护会话应持续维护续接文件，尤其是多提案、多 PR 或长任务。
- `SOP.md`：新增“新窗口续接 / 断点恢复”入口，说明启动顺序与读取文件。
- `README.md`：在版本摘要或使用说明中简述会话续接机制。
- `template-docs/session-handoff.example.md`（建议新增）：提供续接文件模板。

### 可选脚本

- `scripts/session-checkpoint.ps1`（可选）：生成 / 更新续接文件骨架，减少 AI 手写格式漂移。
- `scripts/session-checkpoint.sh`（可选）：如需跨平台 Bash 入口，再补同等能力。

脚本不是第一阶段必需；建议先用规则 + 样例落地，待格式稳定后再脚本化。

## 8. 版本影响

建议作为 `v1.19.0` 或后续 minor 版本落地。

理由：该机制改变 AI 工作流与模板方法论入口，影响规则、提示词、SOP、维护文档与派生项目使用方式，属于可见的方法论增强。

若与 `ai/doc-standards/00-09` 路径迁移同批落地，可合并为同一个 minor 版本；若希望降低 PR 风险，也可拆为独立 patch / minor。

## 9. 验收口径

- 新会话启动规则明确要求读取本地续接文件，并结合 git 状态恢复上下文。
- AI 输出多步骤计划后，规则明确要求自动写入 / 更新续接文件。
- `.gitignore` 同时排除 `NEXT-STEPS.md` 与 `.ai/session-handoff.md`。
- 常用开发 / 维护 prompt 至少覆盖：计划落盘、步骤更新、结束前刷新下一步。
- 样例续接文件结构清晰，包含当前任务、进度、计划、最近改动、下次优先做、阻塞项、恢复命令。
- `check-template.sh` 可增加断言，确保关键规则 / prompt / `.gitignore` 引用了续接机制，防止后续文档滞后。

## 10. 风险与缓解

- **自动写文件与确认规则冲突**：在规则中明确续接文件写入的授权边界；若项目要求所有写入前确认，则 AI 应先说明会自动维护本地续接文件并取得本轮授权。
- **续接文件包含敏感信息**：默认 `.gitignore`；规则要求不得记录 token、密钥、账号、客户敏感数据，只记录任务状态与文件路径。
- **格式漂移**：提供样例文件，后续可用脚本统一生成骨架。
- **AI 忘记更新**：在 `ai/index.md` 引入独立规则，并在常用 prompt 中重复关键触发点；`check-template.sh` 增加稳定关键词断言。
- **多个 AI 工具并行写入冲突**：以 git 工作区实际状态为准；续接文件只作为提示线索，不作为权威事实。
