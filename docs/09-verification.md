# 09 验证计划（Verification）

> AI生成初稿，人工确认。测试策略 + REQ→用例追溯矩阵 + 分阶段验证范围。
> 与 08（“何时做”）正交：本文回答“怎么算对”。按积累式演进（global-rules §8）。
>
> 适用条件：所有项目必备（每个项目都需验证）。无条件保留，不按项目形态省略。

## 1. 测试策略
（单元 / 集成 / 验收；数据夹具；与环境）

## 2. REQ → 用例追溯矩阵
（逐 REQ：验收用例 + 阶段标签 + 状态；当前阶段 REQ 必须有用例）

## 3. 分阶段验证范围
- 当前阶段：覆盖哪些 REQ（对应 02-srs 当前阶段段）
- 升阶段：在原位追加新阶段 REQ 的用例（global-rules §8）

## 最小示例（填写粒度参考）

```markdown
## 1. 测试策略
- 单元测试：覆盖领域处理模块的正常输入、空输入、边界输入。
- 集成测试：覆盖入口到处理结果的完整流程。
- 验收测试：按 REQ 逐条验证用户可观察行为。

## 2. REQ → 用例追溯矩阵
| REQ | 阶段 | 状态 | 用例 ID | 验证方式 | 通过标准 |
|---|---|---|---|---|---|
| REQ-001 | [P1] | P1-已设计 | TC-001 | 自动测试 + 人工验收 | 输入有效文本，返回预期结果 |
| REQ-005 | [P2] | 骨架 | TC-005 | 待 Phase2 细化 | 可查询历史记录 |

## 3. 分阶段验证范围
- Phase1：覆盖 REQ-001；不验证历史记录、登录、导出。
- Phase2：在原位追加 REQ-005 等用例，不删除 Phase1 用例。
```
