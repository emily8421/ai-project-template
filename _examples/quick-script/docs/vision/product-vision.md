# 产品愿景叙事：Meeting Notes Cleaner

> 产品愿景叙事·不直接驱动开发。本文是工程文档的输入，AI 需要先从中抽取轻量文档，再进入开发。

我经常从会议软件、聊天工具或手动记录中得到一份比较乱的会议纪要。它通常包含多余空行、混乱缩进、不同风格的项目符号，以及前后空白。每次分享给别人前，我都要手动整理格式，耗时但没有技术难度。

我希望做一个本地小脚本：输入一个文本文件，输出一份更干净的文本文件。第一阶段只需要做最常见的格式清理：去掉行首行尾空白、合并连续空行、把 `*` / `+` 统一成 `-` 项目符号。

这个工具不需要数据库，不需要 Web 页面，不需要复杂配置，也不需要 AI 模型。它应该足够简单，可以快速实现、容易测试，并且遇到输入文件不存在等错误时给出清楚提示。

未来也许可以支持更多清理规则，比如自动识别标题、提取行动项、导出 Markdown。但这些都不是第一阶段目标。
