# 02 需求规格说明（SRS）

> Phase1 需求必须可验证；小脚本也要明确验收口径。

## Phase1 需求

| ID | 阶段 | 状态 | 需求 | 验证方式 |
|---|---|---|---|---|
| REQ-001 | [P1] | P1-已设计 | 系统必须读取一个输入文本文件。 | 提供存在的输入文件，脚本可读取内容 |
| REQ-002 | [P1] | P1-已设计 | 系统必须把清理结果写入指定输出文件。 | 运行后输出文件存在且内容为清理结果 |
| REQ-003 | [P1] | P1-已设计 | 系统必须去掉每行首尾空白。 | 输入含行首 / 行尾空白，输出对应空白被移除 |
| REQ-004 | [P1] | P1-已设计 | 系统必须把连续两个及以上空行合并为单个空行。 | 输入多空行，输出最多保留一个连续空行 |
| REQ-005 | [P1] | P1-已设计 | 系统必须把以 `*` 或 `+` 开头的项目符号统一为 `-`。 | 输入 `* a` / `+ b`，输出 `- a` / `- b` |
| REQ-006 | [P1] | P1-已设计 | 系统必须在输入文件不存在或参数缺失时返回非 0 退出码并输出可理解错误。 | 错误输入时 stderr 有提示且退出码非 0 |
