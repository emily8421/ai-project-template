# 04 系统架构

## 1. 整体架构图

```text
终端命令
   │
   ▼
CLI 入口（参数解析）
   │
   ▼
文本清洗服务
   │
   ▼
本地文件系统（输入文件 / 输出文件）
```

整体为单进程本地脚本，无网络、无数据库，覆盖 03-PRD 的全部功能范围。

## 2. 模块划分
- `cli/` 命令入口层：解析参数、校验路径、控制退出码
- `service/` 文本清洗层：实现空白裁剪与空行合并
- 文件系统：读取输入文件、写入输出文件

## 3. 技术选型理由
- Python 标准库 `argparse`：足够支持两个参数和错误提示，不需要额外依赖
- 本地文件读写：需求只涉及单文件输入输出，不需要数据库或缓存
- 同步执行：单个文本文件处理量小，没有引入并发或异步的价值
