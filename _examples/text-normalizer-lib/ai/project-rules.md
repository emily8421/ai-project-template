## 0. 项目标识
项目名称：TextNormalizer
代号/缩写：tn

## 1. Phase边界
当前阶段：Phase1

允许：
- Python 3.11 内部库
- 纯字符串输入输出
- 同步实现

禁止：
- 数据库存储
- CLI 命令入口
- HTTP API / Web UI

下一阶段预告：
- Phase2：支持更多可配置规则与批量字符串处理

## 2. 技术栈约束
- 确定：Python 3.11 / 标准库 `re`
- 禁止引入：FastAPI、Flask、Click、Typer、任何数据库驱动

## 3. 项目形态与文档裁剪
- 是否有持久化存储：无
- 是否有对外接口：无
- docs/06-db-design.md：省略
- docs/07-api-spec.md：省略
- 需要保留的代码目录：backend/ tests/

## 4. 目录规范的项目特例
无，遵循 global-rules 通用目录标准。

## 5. 编码约定与禁区

> Phase 级功能禁止见 §1，技术栈替代品禁止见 §2，本节只管代码层。

### 5.1 既有约定（新代码必须向其看齐）
- 命名：Python snake_case（函数/变量/文件）；类 PascalCase
- 分层与目录：`backend/normalizer.py` 暴露公开函数；辅助规则函数保持私有
- 既有模式：公开 API 只接收字符串并返回字符串，不直接读写文件
- 错误处理 / 日志：参数类型不符直接抛 `ValueError`；Phase1 不引入日志框架

### 5.2 禁区（未经人工确认不得触碰）
- 不得擅改的文件 / 模块：`ai/` 通用规则文件
- 不得擅自引入的依赖：任何新依赖须先确认
- 不得自行实现的功能：CLI、HTTP 服务、数据库缓存——均为 §1 禁止项
