# 04 系统架构

> 轻量脚本架构只保留必要模块，避免过度设计。

## 1. 整体架构图

```text
用户命令
  ↓
cli.py（参数解析、文件读写、错误提示）
  ↓
cleaner.py（纯文本清理规则）
  ↓
输出文件
```

## 2. 模块划分

| 模块 | 阶段 | 状态 | 职责 | 对应功能 |
|---|---|---|---|---|
| CLI 层 | [P1] | P1-已设计 | 解析输入 / 输出路径，读取文件，写入结果，处理错误 | F-001/F-003/F-004 |
| 清理规则层 | [P1] | P1-已设计 | 执行去首尾空白、合并空行、统一项目符号 | F-002 |
| 扩展规则层 | [P2] | 骨架 | 支持更多规则和行动项提取 | F-101 |

## 3. 技术选型理由

- 选择单机 CLI：目标是快速清理本地文件，无需服务化。
- 清理规则保持纯函数：便于单元测试，避免文件系统影响业务逻辑测试。
- 不引入第三方依赖：Phase1 规则简单，标准库足够。
