## 0. 项目标识
项目名称：MiniTodo
代号/缩写：mt（用于表前缀；本项目仅 todos 单表，暂未使用）

## 1. Phase边界
当前阶段：Phase1

允许：
- FastAPI + SQLite
- 单人本地运行
- 同步实现

禁止：
- 多用户 / 用户系统 / 鉴权
- PostgreSQL / 其它独立数据库服务
- 云端部署

下一阶段预告：
- Phase2：分类标签、到期时间（仍单人）

## 2. 技术栈约束
- 确定：Python 3.11 / FastAPI 0.110+ / SQLite（标准库 sqlite3）
- 禁止引入：Flask、Django、SQLAlchemy 等 ORM、PostgreSQL 驱动

## 2.5 运行环境与资源约束

- 本机环境文档：示例项目不采集真实机器信息；派生项目需运行 `scripts/collect-env.ps1` 生成 `docs/env/local-env.md`。
- Demo 阶段必须能在本机运行的部分：FastAPI 单进程服务、SQLite 文件、接口测试。
- 允许降级 / Mock / 远程运行的部分：无；Phase1 REST API 可本机运行。
- 禁止在本机运行的重资源部分：PostgreSQL、容器编排、多用户鉴权服务。
- 是否允许使用公司服务器：不需要。
- 若需服务器，资源申请口径：暂不需要；若 Phase2 引入多人协作或独立数据库，再评估 CPU、内存、磁盘、端口与部署权限。

## 5. 编码约定与禁区

> Phase 级功能禁止见 §1，技术栈替代品禁止见 §2，本节只管代码层。

### 5.1 既有约定（新代码必须向其看齐）
- 命名：Python snake_case（函数/变量/文件）；类 PascalCase
- 分层与目录：backend/ 下分 api/（路由）、model/（数据访问）、db.py（连接与建表）；HTTP 处理只在 api 层
- 既有模式：数据访问统一经 model 层的 repo 函数，api 层不直接写 SQL
- 错误处理 / 日志：资源不存在返回 404；入参不符返回 422；Phase1 不引入全局异常中间件

### 5.2 禁区（未经人工确认不得触碰）
- 不得擅改的文件 / 模块：backend/db.py（建表逻辑）；global-rules.md / index.md（通用层，永不动）
- 不得擅自引入的依赖：任何新依赖须先确认（Phase1 目标是零额外依赖）
- 不得自行实现的功能：鉴权、多用户、定时任务——均为 §1 禁止项，点名为禁区
- 愿景 / 01 中的功能点不等于已批准实现；阶段归属以 `docs/03-prd.md` §3 路线图为准，编码以 §1 当前阶段为准

## 6. AI修改确认规则

- AI在进行任何文件新增、修改、删除、重命名、格式化、批量替换前，必须先说明计划、影响范围与预期变更文件，并等待用户明确确认后再执行。
- AI在运行任何可能写入文件、安装依赖、生成构建产物、修改配置或改变项目状态的命令前，必须先询问用户确认。
- 只读分析操作（如读取文件、搜索代码、查看 Git 状态）无需逐次确认，但不得借只读分析之名修改项目内容。
- 用户在单次消息中明确要求“直接修改”“执行修复”“不必确认”等同类授权时，仅对该次明确任务生效；后续新任务仍默认先确认再修改。
