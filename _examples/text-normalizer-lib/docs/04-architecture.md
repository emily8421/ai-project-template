# 04 系统架构

## 1. 整体架构图

```text
调用方模块
   │
   ▼
normalize_text(text)
   │
   ▼
标准库正则与字符串处理
```

整体为单模块内部库，无网络、无数据库、无文件 IO，覆盖 03-PRD 的全部功能范围。

## 2. 模块划分
- `normalizer` 公开函数层：提供可导入调用的唯一公开接口
- 私有规则函数：封装空白清理与参数校验

## 3. 技术选型理由
- Python 标准库 `re`：足够处理连续空白压缩，不需要额外依赖
- 纯函数接口：调用方只关心输入输出字符串，避免把文件 IO 混入库逻辑
- 同步执行：字符串处理开销小，无需异步
