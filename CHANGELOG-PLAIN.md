# CHANGELOG-PLAIN

> Sync notice: This file is maintained by `ai-project-template` and may be overwritten when a derived project syncs template methodology.
> Do not edit it directly in derived projects; propose reusable changes in `_proposals/` and upstream them to the template repository.

本文是 `CHANGELOG.md` 的大白话同步版，按同一版本顺序解释“这版到底改善了什么”。权威版本事实仍以 `VERSION`、`CHANGELOG.md` 和 Git 历史为准；本文件只帮助人快速读懂发布影响。

## v1.56.6（2026-07-23）

把 PR / CI 收尾的速查入口转正。以后 AI 处理 push、开 PR、看 checks、合并、清理分支时，会先读一份短 profile，少翻长规则文件；但高风险远端动作仍然必须先问你确认。

## v1.56.5（2026-07-22）

新增这份大白话版 changelog，并把它加入模板同步范围。以后派生项目同步模板时，除了正式 `CHANGELOG.md`，也能拿到一份更容易读懂的版本说明。

## v1.56.4（2026-07-22）

改善 Windows 大批量同步模板时的操作方法：不要把满屏 CRLF warning 和长文件列表直接塞进上下文，而是用长超时、日志文件和摘要检查来判断是否真有问题。

## v1.56.3（2026-07-22）

补强复杂 Web 项目的主入口文件边界：`App.tsx` 这类主文件只负责组合和调度，业务状态、事件处理和副作用应拆到领域 hook，避免越写越大。

## v1.56.2（2026-07-22）

让模板自检失败更好排查。以前只知道某条断言失败，现在会告诉你失败文件、匹配规则和复现命令，维护者可以直接定位问题。

## v1.56.1（2026-07-22）

让 token hotspot 记录不再只是零散单条。当本地累计多份记录时，AI 会提醒生成或更新汇总，方便发现长期反复浪费上下文的地方。

## v1.56.0（2026-07-21）

新增运行时健康诊断脚本。它不只看 Node 版本对不对，还会看 Node 是从 Volta、nvm、fnm、系统 PATH 还是异常路径来的，帮助判断环境问题根因。

## v1.55.1（2026-07-21）

让前置检查能提醒 Node 主版本漂移。项目声明一个 Node 版本、机器实际跑另一个版本时，会给 warning，但不强制失败。

## v1.55.0（2026-07-20）

建立运行时版本锁定机制。项目可以写清楚要用哪个 Node / Python / Java 版本、用什么工具切换、CI 怎么校验，先用 Node 跑通第一版。

## v1.54.3（2026-07-19）

修复派生项目同步 PR squash 合并后的误报。GitHub 会给 squash 标题加 `(#N)`，现在同步检查能识别这种正常提交格式。

## v1.54.2（2026-07-18）

补充受限网络和代理配置提示。同步模板拉取失败时，不再只提示“可能是私有仓库权限”，也会提示代理和网络排查方式。

## v1.54.1（2026-07-16）

新增母模板侧的派生项目登记表。它不下发给派生项目，只帮助模板维护者知道有哪些派生项目，避免同步和回流时漏掉。

## v1.54.0（2026-07-16）

给存量派生项目增加版本机制迁移提示。检查脚本会判断老项目是否已经启用项目自有版本规则，并给出非阻断迁移建议。

## v1.53.0（2026-07-16）
